## 3. [HKUDS/OpenSpace](https://github.com/HKUDS/OpenSpace)

**1,586 stars this week** · Python

OpenSpace is a shared memory/skill layer that sits on top of existing AI coding agents (Claude Code, Codex, Cursor) so they can learn from past task solutions, skip redundant reasoning, and share successful patterns across agent instances — cutting token usage by ~46%.

**Use case**

The core problem: every time you ask Claude Code or Codex to do something similar to a task you've done before, it starts from zero — burning tokens re-reasoning through the same problem space and making the same exploratory mistakes. For example, if you've already solved 'add Supabase RLS policies for a multi-tenant blog' once, OpenSpace stores that solution pattern and injects it as context the next time any agent hits a structurally similar task, instead of letting the agent stumble through it again.

**Why it's trending**

This dropped right as Claude Code and OpenAI Codex CLI are seeing massive adoption — developers are running agentic coding sessions that cost real money in tokens, and OpenSpace directly attacks that cost problem with a concrete benchmark ($11K earned / 46% token reduction) rather than vague efficiency claims.

**How to use it**

1. Install: `pip install openspace-agent` (Python 3.12+) and run `openspace init` to set up a local skill store.
2. Point it at your existing agent — e.g., wrap a Claude Code session: `openspace --agent claude-code --query 'add Supabase auth middleware to my Next.js API routes'`.
3. OpenSpace retrieves semantically similar past solutions from its vector store before the agent starts, injecting them as condensed context rather than raw history.
4. After the task completes successfully, OpenSpace indexes the solution pattern automatically — subsequent runs (or other agents in the same workspace) benefit immediately.
5. To share skills across a team or between projects, push your skill store to the OpenSpace cloud: `openspace sync --remote https://open-space.cloud/your-workspace`.
